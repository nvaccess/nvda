1. Einführung

NVDA ist ein Bildschirmleser, der es blinden und stark sehbehinderten ermöglicht, Windows-Betriebssysteme zu verwenden.  Indem NVDA das Betriebssystem danach abfragt, welche Fenster auf dem Bildschirm zu sehen sind, welches Steuerelement grade ausgewählt ist und wo Änderungen

passieren, kann NVDA dem Anwender schnell und leicht verständlich über

synthetische Sprache mitteilen, was auf dem Bildschirm vor sich geht. 



NVDA unterstützt (stand März 2007) sapi5-Synthesizer; der modulare Aufbau von NVDA erlaubt die Unterstützung weiterer Sprachausgaben. An einer Unterstützung für sapi4 wird derzeit gearbeitet.  {Anmerkung des Übersetzers: Mittlerweile unterstützt NVDA auch sapi4-Sprachausgaben. Sollten Sie weder über sapi4- noch über sapi5-Sprachausgaben in ihrer landessprache verfügen, können Sie auch den mit NVDA mitgelieferten Synthesizer espeak verwenden. Dieser Synthesizer wird auch bei der Installation verwendet (siehe unten)}



NVDA verwendet eine Mischung aus msaa- und programmspezifischen Objektaufrufen und verzichtet dabei auf Video intercept- oder grafikkartentreiber. Der Vorteil hierbei ist, dass NVDA auf vielen Systemen unabhängig von der Grafikkarte oder vom eingestellten Grafikmodus funktioniert. Außerdem installiert NVDA keinerlei Dateien oder Treiber, die für den Erststart einen Neustart des Systems erfordern. 



NVDA unterstützt momentan noch keine Braillezeilen, es wird jedoch daran gearbeitet.



Für NVDA's meldungen und Benutzeroberfläche sind Übersetzungen in einigen Sprachen verfügbar, weitere kommen in nächster Zukunft hinzu.



NVDA kann Text in vielen Sprachen lesen, solange sie einen sapi5-Synthesizer oder eine espeak-Stimme in der entsprechenden Sprache besitzen.



NVDA befindet sich noch im Entwicklungsstadium. Er ist außerdem quelloffen, was bedeutet, dass jeder den Quellcode herunterladen lesen und sogar ändern kann, um NVDA an seine Bedürfnisse anzupassen. Es kann jedoch vorkommen, dass NVDA fehler enthält oder dass es nicht auf allen Systemen einwandfrei funktioniert. Aber wenn mehr und mehr Anwender NVDA ausprobieren und programmfehler an die Entwickler melden, sollte sich NVDA weiterentwickeln und blinden und sehbehinderten Windows-anwendern eine voll ausgereifte kostenlose Lösung bieten. 



NVDA arbeitet mit vielen Programmen zusammen. wie gut ein Programm mit NVDA harmoniert, hängt davon ab, wie zugänglich dieses Programm ist. 

Von grundlegenden Windows-Funktionen abgesehen harmoniert NVDA mit Wordpad, dem Editor, Outlook Express und Internet Explorer. Er bietet außerdem grundlegende Unterstützung für Microsoft Word 2000/xp/2003, und Microsoft Excel 2000/xp/2003. Mit Office 2007 wurde NVDA noch nicht getestet.



NVDA bietet außerdem umfangreiche Unterstützung für Mozilla Firefox (version 2 und höher). Für optimale Zusammenarbeit mit NVDA ist Firefox sogar der empfohlene Browser.



Diese Anleitung versteht sich nicht als komplettes Handbuch für die Bedienung von NVDA und Windows, bietet jedoch genügend Informationen, um in NVDA einzusteigen und sich dessen Funktionen selbst zu erschließen. 



2. Systemanforderungen

NVDA läuft auf Windows XP und Windows Vista. Die Funktionsfähigkeit von NVDA unter Windows 2000 ist jedoch noch nicht ausreichend getestet.  NVDA funktioniert weder mit Windows 95/98/me noch mit 64-bit-Versionen von Windows. {Anmerkung des Übersetzers: NVDA Release 425 wurde auf

Windows xp x64 erfolgreich getestet und funktioniert stabil.}



für ein reibungsloses Funktionieren von NVDA wird ein Rechner mit

mindestens 1 ghz Taktfrequenz und 256mb Ram empfohlen.  NVDA wird zwar

auch auf langsameren Rechnern funktionieren, Wenn dann allerdings noch

rechenintensive Aufgaben wie das Laden von Webseiten hinzukommen, könnte

das den Rechner und/oder NVDA in die Knie zwingen. 





3. NVDA besorgen

Sie können NVDA von der Webseite 

http://www.NVDA-project.org/ herunterladen.



Auf der Seite "Downloads" finden Sie einen Verweis auf die neueste Version von NVDA.

NVDA releases werden nach Revisionen durchnumeriert. Diese

Revisionsnummer spiegelt wieder, wie oft der Quellcode geändert wurde.

Dass eine Revision auf der Webseite veröffentlicht wurde, bedeutet

jedoch nicht, dass sie komplett fehlerfrei ist. Es bedeutet lediglich,

dass die aktuelle Version von NVDA auf den Systemen der Entwickler

einwandfrei läuft und die Entwickler der Meinung sind, dass NVDA vom

Rest der Welt getestet werden sollte.



Laden Sie das Archiv herunter und entpacken Sie es in ein beliebiges

Verzeichnis auf ihrem System. Sie werden NVDA wahrscheinlich von einer

Festplatte ihres Systems aus starten, es ist jedoch auch möglich, einen

usb-stick oder sogar eine cd-rom als Zielort für NVDA zu verwenden. 

{Anmerkung des Übersetzers: Seit Version 0.5 bietet NVDA neben der

portablen Version auch noch eine sprechende Installation an. Hierbei

wird der ebenfalls quelloffene und freie Sprachsynthesizer espeak

verwendet, der in vielen Sprachen verfügbar ist. Wenn sie das

installationsprogramm starten, wird sie zunächst ein englischsprachiger

Textbaustein darüber informieren, dass sie die Installation beim Drücken

von Eingabe mit Sprachunterstützung fortsetzen können. Ein drücken von

Escape führt zu einer Installation ohne Sprachunterstützung. Obwohl der

Textbaustein englischsprachig ist, erscheint die Meldung auf dem

Bildschirm jedoch in der im Betriebssystem eingestellten Sprache. Wenn

Sie die Installation mit Sprache gewählt haben, wird der NVDA temporär

ausgepackt und gestartet. Außerdem wird espeak in der Sprache

eingebunden, die im System eingestellt ist. Beispiel: Wenn Sie ein

deutsches Windows haben, erscheint der Installationsassistent in Form

deutschsprachiger Dialoge, die von einer deutschen Espeak-stimme

gesprochen werden. Nach der Installation wird der NVDA gestartet und

steht ab sofort über die Tastenkombination strg+alt+n zur Verfügung.} 



4. NVDA starten



Um NVDA zu starten, führen Sie die Datei NVDA.exe im

Installationsverzeichnis aus.



Wenn Sie NVDA starten, hören Sie als erstes eine ansteigende Tonfolge

und anschließend die Meldung "NVDA gestartet". Abhängig davon, wie

schnell ihr Rechner ist und von welcher Art Medium sie NVDA starten,

kann der Start einige Zeit dauern.  Wenn der Start länger dauert, hören

Sie die Meldung "lade Untersysteme, bitte warten...".



Wenn sie stattdessen eine absteigende Tonfolge oder gar den

Windows-fehler-sound hören, ist bei NVDA ein Fehler aufgetreten und sie

sollten den Entwicklern einen Fehlerbericht schicken.  Für nähere

Informationen hierüber sehen sie auf der NVDA-Webseite nach.



Wurde NVDA erfolgreich gestartet, wird die Benutzeroberfläche angezeigt,

in der sie die Einstellungen ändern oder NVDA beenden können. 



5. die Benutzeroberfläche



Das Fenster der NVDA-Benutzeroberfläche enthält eine Menüleiste, mit

deren Menüs sie den NVDA einstellen oder beenden können.  Es enthält

außerdem ein Dokumentfenster, in dem viele häufig benutzte

NVDA-Tastenkombinationen aufgelistet sind.



Die Benutzeroberfläche mit ihrer Menüleiste, dem Dokumentfenster und den

vielen Dialogfenstern ist ein idealer Ort, um die Navigationsfunktionen

von NVDA auszuprobieren.



5.1. anzeigen und verbergen des NVDA-Fensters



Obwohl das NVDA-Fenster beim start des Programms erscheint, muss es

nicht die ganze Zeit im Vordergrund zu sehen sein, während Sie mit NVDA

arbeiten. Es gibt nichts schlimmeres, als zu viele offene

Anwendungsfenster, wenn sie sich mit alt+tab durch die Anwendungen

bewegen, um die richtige zu finden.

  

Um das NVDA-Fenster zu verbergen, drücken Sie alt+f4, während sie sich

innerhalb des NVDA-quickstart-Dokuments befinden. Hierdurch wird NVDA

nicht beendet, sondern nur das Fenster verborgen.



Um das NVDA-Fenster wieder anzuzeigen, können Sie entweder einfügen+n

drücken oder Sie drücken auf dem NVDA-Symbol in der systemablage die

Eingabetaste.





5.2. speichern und Laden der Einstellungen



NVDA speichert seine Einstellungen in einer Datei namens NVDA.ini in

seinem Verzeichnis. Sie werden diese Datei wahrscheinlich nicht

editieren müssen, für erfahrene Anwender bietet sie jedoch die

Möglichkeit Einstellungen zu ändern, für die es in der

Benutzeroberfläche (noch) keine Dialogfelder gibt.



NVDA speichert seine einstellungen nicht automatisch, sodass sie zu

ihren originaleinstellungen zurückkehren, wenn sie das Programm neu

starten.  Um die Einstellungen zu speichern, können Sie entweder im Menü

NVDA den Befehl "Konfiguration speichern" wählen oder strg+s drücken,

während sie sich im NVDA-Fenster befinden.



Wenn sie NVDA auf einem schreibgeschützten Dateisystem gespeichert haben

(wie beispielsweise eine cd), kann er die Einstellungen nicht speichern

und wird sie auch in einem Meldungsfenster diesbezüglich informieren.



Wenn Sie einmal etwas falsch eingestellt haben sollten, können Sie

entweder über den Befehl NVDA/zur gespeicherten Konfiguration

zurückkehren oder durch Drücken der Tastenkombination strg+r die

gespeicherte Konfiguration wiederherstellen.





5.3. einstellungen für die benutzeroberfläche



Dieses Dialogfeld befindet sich im Menü einstellungen.Es erlaubt Ihnen

einerseits die Sprache von NVDA-Dialogen und -Meldungen einzustellen und

andererseits festzulegen, ob das NVDA-Fenster beim Start angezeigt oder

die Konfiguration beim Beenden automatisch gespeichert werden soll.



5.4. sprachausgaben



Dieses Dialogfeld befindet sich im Menü einstellungen ermöglicht die Wahl der zu verwendenden Sprachausgabe.

Momentan werden sapi4 und sapi5 unterstützt. Eine Unterstützung des

quelloffenen und freien Sprachsynthesizers espeak ist ebenfalls

enthalten. 



5.5. Stimmeneinstellungen



Unter einstellungen/Stimmeneinstellungen finden Sie einen Dialog, in dem

Sie NVDA mitteilen können, mit welcher Stimme sie arbeiten möchten. Hier

finden Sie außerdem Einstellmöglichkeiten für Stimmhöhe, Geschwindigkeit

und Lautstärke. 



Die Geschwindigkeit kann außerdem mit den Tasten einfügen+bild-auf und

einfügen+bild-ab geregelt werden.  Bild auf erhöht die Geschwindigkeit

um 5%, bild ab verringert sie um 5%.



Außerdem gibt es noch zwei weitere Optionen: "Satzzeichen sprechen" und

"groß' vor großbuchstaben ansagen". "Satzzeichen sprechen" steuert

hierbei, ob NVDA die Satzzeichen aussprechen oder nur zur Steuerung der

Satzmelodie verwenden soll.  "Groß bei Großbuchstaben ansagen" bestimmt,

ob NVDA das Wort "Groß" vor alleinstehenden Großbuchstaben sagen soll.

Normalerweise wird NVDA bei Großbuchstaben die Stimme anheben. Es gibt

jedoch Sprachausgaben, die dies nicht unterstützen. Für solche

Sprachausgaben kann es sinnvoll sein, diese Option einzuschalten.



5.6. Tastatur



NVDA kann während der Eingabe Zeichen, Wörter und Tastenkombinationen

ansagen. All diese Optionen finden Sie im Dialogfeld "Tastatur" im

Menü einstellungen.



Um diese Einstellungen zur laufzeit zu ändern, benutzen Sie die

Tastenkombinationen einfügen+2 (Zeichen sprechen), einfügen+3 (Wörter

sprechen) und einfügen+4 (Funktionstasten sprechen).

{ Außerden können Sie hier einstellen, welche Tasten als NVDA-Modifikator verwendet werden sollen d.h. mit welchen Tasten Sie NVDA-spezifsche Tastenkombinationen drücken möchten. Standardmäßig sind hier beide Einfügetasten aktiviert. Sie können Hier Caps Lock, Einfügen sowie die Einfügen-Taste des Nummernblocks unabhängig voneinander aktivieren oder deaktivieren. }





5.7. Mauseinstellungen



Sie können NVDA so einstellen, dass er ansagt, wenn sich die Form des

Mauszeigers ändert (so z.B.  wenn ein Programm geladen wird, wenn Sie

die Maus bewegen oder wennsich der mauszeiger in einem eingabefeld

befindet). Außerdem können Sie NVDA so einstellen, dass er den Namen des

objekts ansagt, das sich unter dem Mauszeiger befindet, wenn Sie die

maus bewegen. Diese Optionen finden Sie im Dialogfeld

"Mauseinstellungen" im Menü einstellungen.



5.8. Objektpräsentation



Die Optionen in diesem Dialogfeld bestimmen, wie NVDA mit

windows-Objekten umgeht. In diesem Dialog können sie folgendes

einstellen:

- ob NVDA Tooltips und Hilfesprechblasen vorlesen soll

- ob der Status eines objekts (aktiviert, markiert gedrückt etc) vor dessen Namen angesagt werden soll

- ob mit Hilfe von Signaltönen angezeigt werden soll, dass sich ein Fortschrittsbalken bewegt

{5.9 virtuelle Puffer

In diesem Dialogfeld legen Sie fest, welche Objekte welchen Typs beim Navigieren durch virtuelle Puffer angekündigt werden sollen. Wenn Sie beispielsweise "tabellen ansagen" aktiviert haben und beim Navigieren durch virtuelle Puffer auf eine Tabelle treffen, so wird "tabelle mit X Spalten und y Reihen" angesagt. Ist die option nicht aktiv, so wird stillschweigend in die Tabelle gewechselt. Derartige Objekttypansagen können hier für viele html-Objekte wie z.B.  Absätze, Formulare, Formularfelder, Verweise, Aufzählunglisten etc. aktiviert werden. Weitere Informationen zu virtuellen Puffern finden Sie im Abschnit 7



5.10 Dokumentformatierungen

mit Hilfe dieses Dialogs können Sie die Redseeligkeit von NVDA beim Navigieren durch Dokumente beeiflussen. Hier kann Z.B. aktiviert werden, ob Schriftart, Schriftgröße oder Schriftattribute angesagt werden sollen. }





6. mit NVDA navigieren



6.1. navigieren mit dem Fokus



die gebräuchlichste Art, sich innerhalb von Windows zu bewegen ist die

Verwendung von Standard-Tastenkombinationen wie tab, umschalt+tab,

Pfeiltasten um zwischen Objekten zu wechseln oder alt+tab, um zwischen

Anwendungen umzuschalten. Wenn Sie auf diese Weise navigieren, wird NVDA

ansagen, welches Objekt den Fokus hat.



Erhält ein Objekt den Fokus, so wird NVDA dessen Namen, Typ, Wert,

Beschreibung, Tastenkombination und Position angeben. Die

Objekttypbezeichnungen können sich jedoch von denen anderer

Bildschirmleser unterscheiden.



6.2. objektorientierte Navigation



Wenn sie sich im Betriebssystem umsehen wollen, ohne den Fokus zu

bewegen, können Sie die Objektorientierte Navigation von NVDA verwenden.

Dies ermöglicht Ihnen, sich in einer baumartigen Struktur zwischen den

Objekten zu bewegen.



Die Baumstruktur beginnt mit dem desktop, dem die einzelnen laufenden

Anwendungen untergeordnet sind. Die Anwendungsfenster können ebenfalls

Unterfenster enthalten.



Die Baumstruktur kann bei manchen Anwendumgen sehr umfangreich werden,

dennoch werden Sie aufgrund der logischen anordnung der Objekte die

benötigten Informationen schnell finden.  Dies repräsentiert weniger die

Anordnung der Objekte auf dem Bildschirm als vielmehr die logische

Anordnung innerhalb des Programms. Es wird zwar eine Weile dauern, bis

sie sich eingearbeitet haben,  sie werden sich aber nach einer gewissen

Zeit schnell und einfach durch die objekthierarchie bewegen können.



NVDA bietet im Moment keine Möglichkeit, Bildschirmorientiert zwischen

den Objekten zu navigieren. Dies ist dem Umstand geschuldet, dass sich

NVDA nicht in die Grafikdarstellung hineinhängt und keinen

Intercept-Treiber besitzt.  Trotzdem sollten Sie ein logischeres

Verständnis über den Aufbau einer Anwendung erlangen, als wenn man

versucht, den Bildschirminhalt abzubilden.



Nicht alle anwendungen haben eine perfekte Baumstruktur; es liegt jedoch

am Entwickler jeder einzelnen Anwendung.  die meisten Anwendungen

scheinen jedoch eine ordentliche Baumstruktur zu haben.



Um zwischen den Objekten zu navigieren, verwenden Sie die folgenden

Tasten: 

aktuelles Objekt ansagen: einfügen+5 (Zehnertastatur) 

zum übergeordneten objekt wechseln: einfügen+8 (Zehnertastatur) 

zum vorigen Objekt auf gleicher Ebene wechseln: einfügen+4 (Zehnertastatur) 

zum nächsten Objekt bewegen: einfügen+6 (Zehnertastatur) 

zum ersten untergeordneten Objekt wechseln: einfügen+2 (Zehnertastatur) 

zum Fokus bewegen: einfügen+Minus (Zehnertastatur) 

aktuelles Objekt aktivieren: einfügen+Eingabe (Zehnertastatur) 

die Maus zum aktuellen Objekt bewegen: einfügen+Schrägstrich (Zehnertastatur) 

zur Maus bewegen: einfügen+Stern (Zehnertastatur) 

Anmerkung: Die Verwendung des Ziffernblocks erfordert, dass Numlock

ausgeschaltet ist. 



6.3. Text im aktuellen Objekt lesen



manchmal möchten Sie vielleicht wissen, wie ein bestimmtes Wort

geschrieben wird oder sie möchten eine bestimmte Passage wortweise

lesen. Um den Text innerhalb eines Objekts zu lesen, können Sie die

tasten des Nummernblocks verwenden: zur ersten Zeile gehen: umschalt+7

zur vorigen Zeile gehen: 7 aktuelle Zeile vorlesen: 8 zur nächsten Zeile

gehen: 9 zur letzten Zeile gehen: umschalt+9 voriges Wort lesen: 4

aktuelles Wort lesen: 5 nächstes Wort lesen: 6 zum Zeilenanfang

springen: umschalt+1 voriges Zeichen lesen: 1 aktuelles zeichen lesen: 2

nächstes zeichen lesen: 3 zum Zeilenende springen: umschalt+3 den

gesamten text lesen: Plus Anmerkung: Um die Tasten des Nummernblocks zu

verwenden, muss Numlock ausgeschaltet sein. Normalerweise wird der text

innerhalb eines objekts aus dessen Namen, Wert und Beschreibung

zusammengesetzt. Für Objekte, die tatsächlich text enthalten, wird

wirklich der in einem Objekt enthaltene Text gelesen. Dies gilt zum

Beispiel für eingabefelder. Beispielsweise wird ein Eintrag eines

Listenfeldes Text enthalten, der sich aus dessen namen, Wert und

Beschreibung zusammensetzt. Bei einem Eingabefeld oder einem Dos-fenster

wird der tatsächlich enthaltene Text ausgegeben.



6.4. Mit dem Systemcursor navigieren



Wenn ein Eingabefeld den Fokus hat, können Sie sich mit den üblichen

Tasten durch das Eingabefeld bewegen und den Text editieren. NVDA wird

hierbei ansagen, wenn Sie sich durch das Eingabefeld bewegen, text

markieren oder die Markierung aufheben. 

Wenn sie sich innerhalb eines Eingabefeldes bewegen, wird auch der

NVDA-interne Cursor mitgezogen (siehe voriger Abschnitt).

Sie können auch einfügen+pfeil ab verwenden, um sich den gesamten Inhalt

eines objekts vorlesen zu lassen. Im Unterschied zu numplus wird  hier

jedoch der systemcursor mitgezogen. 



6.5. Navigieren mit der Maus



NVDA kann so eingestellt werden, dass es bei Mausbewegung das Objekt liest, über dem sich die Maus befindet.

Auf diese Art können auch blinde Computerbenutzer eine Eindruck vom Aufbau des Bildschirminhalts bekommen, indem sie die Maus benutzen.  Im Moment kann NVDA nur das Objekt lesen, das sich unter der Maus befindet; eine Navigation mit Hilfe des Mauszeigers ist dagegen noch nicht implementiert.



7. virtuelle Puffer



Komplexe Dokumente wie Webseiten werden in NVDA innerhalb virtueller

Puffer präsentiert.



Ein virtueller Puffer ist eine textliche Repräsentation eines Dokuments,

die ihnen erlaubt, sich mit den Pfeiltasten durch das Dokument zu

bewegen oder bestimmte Formularfelder zu lesen. Von der Navigation mit

den Pfeiltasten einmal abgesehen können sie auch Eingabe oder die

leertaste auf einem Formularfeld drücken, um es zu aktivieren. Dies gilt

für Links, Kontrollfelder, Schaltflächen und auswahlschalter. Drücken

Sie die Leertaste auf einem Kombinationsfeld, so können Sie einen

Eintrag auswählen.  Wenn sie sich auf einem solchen formularfeld

befinden, wird NVDA dessen Typ und Namen ansagen, auch wenn diese

Informationen nicht im virtuellen Puffer angezeigt werden. Dies hat zur

Folge, dass Sie sich trotz der Zusatzinformationen normal durch den text

bewegen können. 



Im Moment verwendet NVDA virtuelle Puffer sowohl für Internet Explorer

als auch für Mozilla Firefox. 

Virtuelle Puffer haben die Fähigkeit,

ausdrücklich aktualisiert zu werden, im Augenblick macht jedoch nur

Firefox von diesem Leistungsmerkmal gebrauch. Obwohl im Internet

Explorer die meisten Webseiten korrekt angezeigt werden, kann Firefox

besser mit seiten umgehen, die sich z.B. mit Hilfe von Javascript

dynamisch aktualisieren. 



Aufgrund der Komplexität virtueller Puffer kann es eine Weile dauern,

bis NVDA eine Webseite geladen hat. die meisten Webseiten werden zwar

schnell geladen, es gibt jedoch auch Seiten, die 5 oder mehr Sekunden

zum Laden benötigen. 



Um eine schnellere Navigation zu ermöglichen, stellt NVDA Schnelltasten

für das springen zu Objekten eines bestimmten Typs zur Verfügung.



Ohne irgendwelche Umschalttasten gedrückt springen sie zum nächsten

objekt, mit Umschalt springen sie zum vorigen Objekt.



Tasten:

Überschrift: h

Listenfeld: l

Listeneintrag: i

Tabelle: t

Verweis: k

Absatz: p

Formularfeld: f





manchmal möchten Sie vielleicht Formulare ausfüllen, ohne dass die

Schnelltasten des virtuellen Puffers im weg sind. Der Modus "virtuellen

Puffer durchlassen" löst genau dieses Problem und kann mit

einfügen+leertaste umgeschaltet werden.



Generell gilt: Wenn sie auf einem Formularfeld Enter drücken, wird der

Modus aktiviert. Wenn die Seite neu geladen wird, wird er wieder

ausgeschaltet.



8. Signaltöne für Fortschrittsanzeigen



Fortschrittsbalken dienen dazu, anzuzeigen, wie weit eine Aufgabe in

windows fortgeschritten ist. Das gilt beispielsweise für das Kopieren

von dateien im Windows-Explorer, für das Laden von Webseiten im Internet

explorer oder für das Prüfen auf neue E-Mails mit Outlook Express.  NVDA

kann durch Signaltöne anzeigen, wie weit ein Fortschrittsbalken

ausgefüllt ist. Je höher der Signalton wird, desto näher rückt der

Fortschrittsbalken der 100%-Marke. Sie werden schnell ein gefühl dafür

entwickeln, wie 0% oder 100% klingen und so sagen können, wie weit ein

Fortschrittsbalken ausgefüllt ist. 



9. Signaltöne als Sprachausgabe



In diesem Modus wird NVDA Signaltöne ausgeben, anstatt den

Bildschirminhalt zu sprechen.  Dies ist beispielsweise sinvoll, wenn Sie

Programme einsetzen, die den Bildschirminhalt sehr oft aktualisieren.

Beispiele hierfür sind cd-Brennprogramme für dos oder das Kopieren von

Dateien in einem Dos-Fenster.  Anstatt jeden Dateinamen oder jedes

fortschreitende Prozent beim Brennen zu hören und die Ansage mit strg

abzubrechen können Sie einfügen+s drücken, um in den Modus "Signaltöne

als Sprachausgabe" umschalten. Alles was sie dann noch von NVDA hören,

sind kurze, hohe Pieptöne. mit einfügen+s schalten Sie wieder zurück auf

die normale sprachausgabe.
