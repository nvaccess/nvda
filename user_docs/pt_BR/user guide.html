1. Introdução

1.1. Características Gerais

O feedback fornecido pelo NVDA através de voz sintética permite às pessoas cegas acessarem e interagirem com o sistema operacional Windows e diversas aplicações de terceiros.

As características principais são:

·         Permite a sua execução diretamente através de dispositivos USB ou de outras mídias portáteis sem necessitar de instalação;

·         Instalação com voz e fácil de usar;

·         Navegar na Web com o Mozilla Firefox 3;

·         Trabalha com o programa de gestão de correio eletrónico Mozilla Thunderbird 3;

·         Suporte precário para o Microsoft Internet Explorer;

·         Suporte básico para o Microsoft Outlook Express / Windows Mail;

·         Suporte básico para o Microsoft Word e Excel;

·         Suporte para aplicações Java acessíveis;

·         Suporte precário para o Adobe Reader;

·         Suporte precário para o IBM Lotus Symphony;

·         Suporte para a Prompt de Comando do Windows e aplicações de console;

·         Anúncio automático do texto por baixo do mouse e indicação em áudio opcional da posição deste periférico;

·         Suporte para quaisquer linhas Braille que sejam controladas através dos drivers de Braille da Freedom Scientific ou do BRLTTY.

1.2. Internacionalização

É importante que as pessoas em qualquer parte do mundo, independentemente do idioma que falem, obtenham o mesmo acesso às tecnologias. O NVDA está traduzido em mais de 20 idiomas incluindo: o Português do Brasil, Croata, Checo, Finlandês, Francês, Galego, Alemão, Húngaro, Italiano, Japonês, Português, Russo, Eslovaco, Espanhol, Tailandês, Chinês Tradicional, Ucraniano, Holandês Sul-Africano, Polaco e Vietnamita.

1.3. Suporte De Sintetizador De Voz

além de fornecer mensagens e interface em diversos idiomas, o NVDA também pode permitir ao usuário ler o conteúdo em qualquer língua, tendo em conta que deverá dispor de um sintetizador de voz que possa falar esse idioma em particular.

O NVDA vem com o eSpeak incluído, um sintetizador de voz livre, open-source e multi-idioma. Adicionalmente, este leitor de telas pode usar ambos os motores de voz SAPI4 e SAPI5 para permitir a saída de voz, bem como com o sintetizador de voz Audiologic.

1.4. Suporte de Braille

Para os usuários que têm uma linha Braille, o NVDA pode transmitir a informação neste sistema de leitura e escrita. Este leitor de telas suporta as linhas Braille Pacmate e Focus da Freedom Scientific, Handy Tech e outras suportadas pelo pacote do BRLTTY se estiverem disponíveis.

1.5. Inovação E Experimentação

O NVDA não se encontra limitado pela necessidade de seguir tendências e demandas do mercado. Os Leitores de telas comerciais têm boas características, mas algumas vezes estão dependentes do que o mercado pretende, em vez das mesmas serem realmente úteis para o usuário. O NVDA poderá não ser sempre tão estável como outros leitores de telas são, mas poderá certamente servir como uma ferramenta experimental e de teste. Pode experimentar facilmente e rapidamente novas ideias e características que talvez não tenham sido encontradas em outros Leitores de telas para o Sistema operacional Windows. Um exemplo disso é a capacidade do NVDA utilizar beeps para indicar ao usuário que uma Barra de progressão está se movendo . O beep mais alto e agudo indica que a mesma está chegando ao fim. As pessoas adicionaram esta característica a outros leitores de telas através de scripts personalizados e similares, mas nunca foi oficialmente adoptada completamente no núcleo de nenhum outro leitor de telas para o Windows.

1.6. Arquitetura e Implementação

O NVDA é escrito na linguagem de programação Python, é construído com uma arquitetura modular e muito do seu código pode ser adicionado e estendido facilmente para dar suporte a novos programas ou a controles no Windows. As App Modules podem ser escritas para se adicionar suporte total a uma aplicação específica, os Buffers Virtuais podem ser escritos para permitirem que o NVDA mostre documentos complexos ou outros dados e os Objetos deste leitor de telas podem ser escritos para se adicionar suporte a controles específicos ou janelas. O NVDA tenta tornar constantemente os controles e elementos de um programa ou Sistema operacional como acessíveis sempre que lhe seja possível, para que o usuário possa procurar ativamente toda a informação desejada . Este leitor de telas não se encontra sujeito a características especiais tal como filtrar e anunciar informação particular que o usuário pretenda saber.

Para comunicar com o Sistema operacional e programas, o NVDA usa o conjunto de funções do Windows, Microsoft Active Accessibility (MSAA), IAccessible2, Java Access Bridge e interfaces de programação específicas fornecidas por determinadas aplicações. Este leitor de telas não utiliza quaisquer drivers específicos de Interceptador de Vídeo ou display hooks e tentará adquirir sempre que possível o maior número de informações a partir das interfaces de programação adequadas, antes de recorrer a outro método.

2. Requerimentos Do Sistema

·         Sistemas operacionais: Todas as versões 32-bit do Windows XP e do Windows Vista (incluindo as versões Server), pode trabalhar parcialmente no Windows 2000 e em versões 64-bit do Windows XP.

·         Memória: 256 mb ou mais de RAM.

·         Velocidade do processador: 1.0 GHz ou superior.

·         Aproximadamente 50 MB de espaço livre.

3. Baixando O NVDA

Se não tiver uma cópia do NVDA, pode fazer o download em: http://www.nvda-project.org/

Entre na seção download e procure o link para baixar a última versão deste leitor de telas .

Atualmente, existem duas formas em que o NVDA é distribuído. Uma é um Instalador que falará através do processo de instalação esta opção permite ao usuário executá-lo a qualquer momento a partir do Menu Iniciar. O outro é um arquivo zip que contém a versão completa portátil do NVDA, que pode ser executada a partir de qualquer lugar, incluindo-se os dispositivos USB ou CD.

4. Instalando O NVDA

4.1. O Instalador Do NVDA

Se já tiver o instalador do NVDA, pressione simplesmente Enter ou um duplo clique esquerdo no arquivo e o mesmo será iniciado .

Enquanto a instalação estiver sendo carregada, ouvirá a música de instalação do NVDA. Após estar carregado, uma cópia temporária deste leitor de telas permitirá que prossiga pelas telas do instalador e instale o NVDA. Note que se estiver executando outra cópia do nvda no momento em que iniciou o instalador, uma mensagem aparecerá avisando que a sua cópia anterior do NVDA será encerrada. Quando pressionar Enter nesta mensagem, o instalador concluirá a mesma e iniciará a sua própria cópia. Isto é necessário para que o instalador possa atualizar adequadamente qualquer arquivo modificado. , se já tiver outra cópia do NVDA instalada, o instalador desinstalará a mesma automaticamente e iniciará a instalação da  nova versão.

Uma vez que tenha completado corretamente a instalação, o instalador alertará que o processo foi concluído e perguntará se deseja reiniciar o sistema. Isto é muito importante para que não hajam consequências desagradáveis.

4.2. O arquivo Zip Portátil

Se tiver a versão do arquivo zip do NVDA, então tudo o que tem de fazer é descompactá-lo para qualquer lugar que deseje no seu disco local ou em outra mídia portátil.

5. Iniciando o NVDA

5.1. executando O NVDA

Se tiver o NVDA instalado , então iniciar o NVDA é realmente simples basta pressionar Control+Alt+N, ou escolher o NVDA a partir do Menu Iniciar, programas, submenu NVDA.

Para iniciar a versão do arquivo zip portátil, vá para a pasta onde descompactou o NVDA e pressione Enter ou um duplo clique esquerdo em nvda.exe.

Assim que o NVDA for executado , ouvirá primeiramente um conjunto de tons ascendentes (informando que o mesmo está sendo carregado). Então deverá ouvir a  mensagem "NVDA ativado". Dependendo da rapidez do seu computador, ou se estiver executando este leitor de telas através de um dispositivo USB ou de outra mídia mais lenta, poderá demorar un pouco para iniciar. Se o NVDA estiver demorando um tempo maior do que o previsto, o leitor de tela dirá "carregando componentes. Por favor aguarde..."

Se não ouvir nada disto, ou escutar um som de erro do Windows, ou um conjunto de sons descendentes, então isto significa que o NVDA tem um erro, e necessitará possivelmente de comunicar o bug aos programadores. Por favor procure na página do NVDA para saber qual o procedimento.

Quando o NVDA for iniciado pela primeira vez, será apresentada uma caixa de diálogo que lhe fornece informações básicas sobre a tecla modificadora do NVDA e o menu deste leitor de tela. (Por favor consulte as seções seguintes para maiores informações acerca desses tópicos). A caixa de diálogo contém também duas caixas de verificação. A primeira permite-lhe controlar se o NVDA utiliza o capslock como uma tecla modificadora do NVDA e a segunda permite configurar se este diálogo de boas vindas será exibido sempre que o NVDA for iniciado.

5.2. Comandos Do Teclado Do NVDA

Muitos dos comandos específicos do teclado deste leitor de tela consistem normalmente no pressionar da tecla modificadora do NVDA em conjunto com uma ou mais outras teclas. Uma exceção a isto são os comandos que servem para se rever o texto que utilizam somente o teclado numérico.

Por padrão o Insert do teclado numérico e o Insert do bloco intermédio podem ser utilizados como teclas modificadoras do NVDA. Contudo, poderá configurar também este leitor de tela para que a tecla CapsLock também se torne numa tecla modificadora do NVDA. Quando isto estiver configurado , pressionar ou manter pressionado o CapsLock faz com que este funcione como uma tecla modificadora do NVDA mas ao pressionar duas vezes rapidamente faz com que funcione como a tecla CapsLock normal.

Muitas das teclas de comando são mencionadas ao longo deste guia do usuário mas uma maneira fácil para explorá-las é ativar a ajuda do teclado.

Para ativar a ajuda do teclado, pressione NVDA+1. Para desativar, tecle NVDA+1 novamente. Enquanto estiver na ajuda do teclado, ao pressionar qualquer tecla o leitor de tela anunciará o que faz (se tiver alguma ação associada). As teclas não executarão a sua função enquanto estiver no modo de ajuda do teclado, desta forma poderemos pressionar as teclas que desejarmos .

6. O Menu Do NVDA

O menu do NVDA permite-lhe controlar as configurações deste leitor de tela, Acessar à ajuda, salvar/restaurar a configuração e sair do NVDA.

6.1. Acessando o Menu Do NVDA

Se este leitor de tela estiver em execução, podemos Acessar seu menu a partir de qualquer lugar do Windows pressionando NVDA+N. Podemos Acessar também o menu do NVDA através da área de notificação do Windows. Basta clicar com o botão direito no ícone do NVDA que lá se encontra, ou mover-se para a área de notificação pressionando a tecla com o logotipo do Windows + B, andar com a Seta para baixo, até o ícone deste leitor de tela e pressionar a Tecla das Aplicações que se encontra em diversos teclados antes do Control direito. Quando o menu for exibido, poderá usar as setas direcionais para navegar pelo menu e ativar um item com a tecla Enter. O menu Preferências permite-lhe configurar o NVDA ao seu gosto, o menu Ferramentas contém uma ou duas ferramentas úteis como o visualizador de registro do NVDA e o console do Python para programadores e o menu Ajuda permite-lhe Acessar o guia do usuário, guia rápido de teclas de atalho, e muito mais. O menu do NVDA também contém itens que lhe permitem salvar ou restaurar a sua configuração. Por fim, existe uma opção que lhe permite sair do NVDA, apesar de conseguir fazer isto com mais eficiência se pressionar NVDA+Q.

As opções que se encontram no menu preferências também podem ser acessadas através de teclas de atalho, bem como as opções salvar e restaurar a configuração que já foram abordadas neste capítulo. Para maiores informações, acesse por favor o guia rápido de teclas de atalho do NVDA.

6.2. Configurando O NVDA

6.2.1. O Menu Preferências

6.2.1.1. Opções Gerais

A caixa de diálogo das Opções gerais encontra-se no menu Preferências e também pode ser acessada pressionando NVDA+Control+G. Para Salvar as opções depois de modificar as configurações , pressione o botão OK. Para cancelar, pressione o botão cancelar ou a tecla escape. A mesma contém as seguintes opções:

Idioma

Uma caixa de combinação que lhe permite selecionar o idioma da interface do Usuário e mensagens do NVDA. Existem diversos idiomas, contudo, a opção padrão é "User Default, Windows". Esta opção informa ao NVDA para utilizar o idioma que o Windows tem definido atualmente (nas Opções Regionais e de Idioma no Painel de Controle). Observe por favor que o NVDA precisa ser reiniciado. Quando escolher o idioma, este leitor de tela perguntará se deseja reiniciar para que a alteração seja efetuada. Pressione OK e o NVDA irá reiniciar.

Salvar A Configuração ao Sair

Uma caixa de verificação que quando está marcada informa ao leitor de tela para Salvar automaticamente a configuração atual quando sair.

Alertar antes de sair do NVDA

Uma caixa de verificação que lhe permite escolher se deseja que apareça um diálogo quando fechar o NVDA com a pergunta se deseja sair. Quando marcada, o diálogo aparecerá quando tentar fechar o NVDA perguntando se deseja sair.

Nível de registro

Uma caixa de combinação que lhe permite selecionar a quantidade de informação que o NVDA coloca no registro quando estiver em execução. Geralmente, os Usuários não necessitam alterar isto já que não é muito amigável. Contudo, se pretender adicionar informação ao relatório de bugs etc., então será importante ajustar para um nível mais elevado.

6.2.1.2. Seleção de Sintetizador

O diálogo do Sintetizador, pode ser encontrado no item "Sintetizador..." no menu Preferências ou pressionando NVDA+Control+S, permitindo-lhe selecionar qual é o sintetizador que o NVDA utilizará para falar.

Esta caixa de diálogo exibe uma caixa de combinação simples que apresenta todos os sintetizadores disponíveis. Selecione o sintetizador que deseja utilizar com as setas e por fim pressione ok. Se existir um erro ao carregar o sintetizador, uma caixa de mensagem aparecerá alertando para este fato e o sintetizador anteriormente carregado será mantido.

A lista atual de sintetizadores que o NVDA suporta é: eSpeak, SAPI4, SAPI5, Audiologic, Display, e Silence.

O sintetizador eSpeak está compilado diretamente com o NVDA e não depende de quaisquer outros drivers especiais ou bibliotecas SAPI para ser instalado. Este leitor de tela inicia utilizando o eSpeak por predefinição. O sintetizador em questão trabalhará em qualquer sistema em que o NVDA trabalhe e funcionará definitivamente quando executar este leitor de tela através de uma unidade de memória USB ou CD, ou em qualquer outro sistema.

A opção SAPI4 permite-lhe usar sintetizadores SAPI. As vozes que estão disponíveis no item SAPI4 incluem as do Eloquence, bem como outras SAPI4 que tenha instaladas no sistema.

O sintetizador SAPI5 permite-lhe usar vozes da Microsoft que estão incluídas na sua máquina. No XP, a voz Sapi5 predefinida é o Microsoft Sam. Já no Vista, a voz padrão é a Microsoft Anna. Dependendo de quais vozes tem instaladas, outras poderão aparecer no item SAPI5.

O sintetizador Display é útil para programadores com visão que pretendam ler o que é que o NVDA está anunciando . Desta forma, podem testar o NVDA em suas aplicações.

O driver do sintetizador Silence não é útil, a não ser que não deseje ter nenhuma voz todas as vezes que executar o NVDA.

Por favor, Observe que não importam quantas vozes diferentes SAPI4 ou SAPI5 (ou motores) tem instalados no seu sistema, somente os sintetizadores SAPI4 e SAPI5 atuais serão apresentados no diálogo dos sintetizadores. Para selecionar um dos motores (vozes), seleccione ou o SAPI4 ou o SAPI5 e então, no diálogo das Opções de voz, escolha a voz desejada .

6.2.1.3. Opções de Voz

O diálogo das Opções de voz encontra-se no menu Preferências ou pode ser acessado através de NVDA+Control+V e contém opções que lhe permitem alterar o som da voz. Pode também configurar diversas opções a partir de qualquer local pressionando NVDA+Control juntamente com uma das setas. As Setas Esquerda e Direita movem-no através de várias opções e as Setas Acima e Abaixo aumentam ou diminuem essa opção respectivamente.

Esta caixa de diálogo contém as seguintes opções:

Voz

A primeira opção que se encontra neste diálogo é uma caixa de combinação que lista todas as vozes do sintetizador atual que tem instalado. Pode utilizar as setas para ouvir os diversos itens. As setas Acima e Esquerda sobem pela lista, enquanto as Setas Abaixo e Direita descem pela lista.

Variante

Se estiver utilizando o eSpeak que acompanha o NVDA, esta é uma caixa de combinação que lhe permite selecionar a Variante do sintetizador com a qual deveria falar. As variantes do eSpeak são parecidas às vozes, mas proporcionam atributos ligeiramente diferentes da voz principal. Algumas variantes são semelhantes a homens, mulheres e algumas parecem-se com sapos.

Velocidade

Esta opção permite-lhe alterar a velocidade da voz. Contém uma barra de controle que lhe possibilita andar a partir do 0 até 100, (sendo 0 a velocidade mais lenta e 100 a mais rápida).

Entoação

Esta opção permite-lhe alterar a entoação da voz. Contém uma barra de controle que lhe possibilita andar a partir do 0 até ao 100, (sendo o 0 a entoação mais grave e 100 a mais aguda).

Volume

Esta opção é uma barra de controle que lhe permite andar a partir do 0 até 100, (sendo o 0 o volume mais baixo e 100 o mais alto).

Inflexão

Uma barra de controle que lhe permite escolher quanta inflexão (subida e descida na entoação) que o sintetizador usaria para falar. (Somente o eSpeak fornece esta função por padrão ).

Falar toda a pontuação

Esta opção contém uma caixa de verificação que quando marcada informa ao NVDA para anunciar todos os símbolos de pontuação como palavras (muito útil para testes de leitura). Quando desmarcada, o NVDA deixa a pontuação sem alterá-la, desta forma, o sintetizador lerá frases com a inflexão correcta. Contudo, os símbolos não serão falados em voz alta.

Aumentar Entoação Para Maiúsculas

Se esta opção estiver selecionada, aumentará a entoação nas maiúsculas.

Anunciar "Maiúscula" Antes De Letras Maiúsculas

Esta opção é uma caixa de verificação que quando marcada informa ao NVDA para anunciar a palavra "maiúscula" antes de qualquer letra maiúscula, quando se anda sobre a mesma ou anuncia-a quando está sendo escrita. Normalmente, o NVDA aumenta ligeiramente a entoação para qualquer letra maiúscula, mas alguns sintetizadores não suportam corretamente esta configuração, assim, esta opção poderá ser utilizada.

Beep para maiúsculas

Se esta caixa de verificação estiver marcada, o NVDA produzirá um pequeno beep cada vez que estiver anunciando um Caracter maiúsculo. Tal como a caixa de verificação 'anunciar maiúscula antes de letras maiúsculas', esta é útil para sintetizadores que não possam alterar a sua entoação para letras maiúsculas.

6.2.1.4. Opções de Braille

A caixa de diálogo das Opções de Braille pode ser acessada através do menu Preferências, item Opções de Braille.

Linha Braille

A primeira opção que encontrará no diálogo das Opções de Braille é uma caixa de combinação que se denomina "Linha Braille". Esta aparecerá com três opções. Desloque-se pelas mesmas com as Setas.

O item sem Braille significa que não está utilizando o Braille.

A opção Freedom Scientific é para as linhas Braille da Freedom Scientific Focus/Pacmate. Atente que esta opção aparece somente se tiver instalado o driver de Braille da Freedom Scientific. Consulte a página oficial dos drivers de Braille desta empresa para mais informações sobre como obter e instalar os drivers.

O item Handy Tech é para as linhas Handy Tech. Observe que esta opção só aparecerá se tiver instalado o driver universal Handy Tech.

O BRLTTY permite-lhe utilizar uma vasta lista de linhas Braille. Por favor, instale a versão para Windows do BRLTTY para ter suporte para estas linhas.

Tabela de Tradução

A opção seguinte que se encontra neste diálogo é uma caixa de combinação para as tabelas Braille. Nesta caixa de combinação, encontrará tabelas Braille para diversos idiomas. Pode-se mover de tabela Braille em tabela Braille na lista utilizando as Setas.

Expandir Para Braille De Computador Palavras Sobre O Foco

Esta opção permite que a palavra que está sobre o cursor apareça em Braille de computador não contraído.

Velocidade de intermitência do Cursor

Esta opção é um campo numérico que lhe permite alterar a velocidade de intermitência do cursor em milésimos de segundo.

Tempo de Permanência de Mensagens (segundos)

Esta opção é um campo numérico que controla durante quanto tempo são apresentadas as mensagens do sistema na linha Braille.

Miscelânea

Quando pressionar NVDA+Control+T, pode controlar o que vai ser apresentado na linha Braille. As duas opções são foco e revisão. O foco mostra o que vai ser acessado através das setas e da tecla tab, em suma, navegação com o foco. A revisão permite-lhe rever o que está na tela, ou seja, navegação de objetos. Pode mover-se rapidamente entre estas opções se pressionar control+NVDA+T.

6.2.1.5. Opções do Teclado

Esta caixa de diálogo encontra-se no menu Preferências, no item "Opções do teclado..." e pode ser acessada pressionando NVDA+Control+K. Contém as seguintes opções:

Layout de teclado

Esta caixa de combinação permite-lhe escolher qual é o tipo do layout do teclado que o NVDA utilizará. Atualmente, os dois que vêm com o NVDA são o Desktop (computador de mesa) e o Laptop (computador portátil).

Utilizar o CapsLock como uma tecla modificadora do NVDA

Se esta caixa de verificação estiver marcada, o CapsLock pode ser usado como uma tecla modificadora do NVDA.

Utilizar Insert do bloco intermédio como uma tecla modificadora do NVDA

Se esta caixa de verificação estiver marcada, a tecla Insert do bloco intermédio (encontra-se normalmente acima das setas, perto do Home e End) pode ser usada como uma tecla modificadora do NVDA.

Utilizar Insert do teclado numérico como uma tecla modificadora do NVDA

Se esta caixa de verificação estiver marcada, a tecla Insert do teclado numérico poderá ser utilizada como uma tecla modificadora do NVDA.

Falar Caracteres digitados Uma caixa de verificação que quando marcada informa que o NVDA anunciará todos os caracteres que escrever no teclado. Pode também configurar esta opção a partir de qualquer local pressionando NVDA+2.
Falar Palavras digitadas Uma caixa de verificação que quando está marcada informa que o NVDA anunciará todas as palavras que escrever no teclado. Pode também configurar esta opção a partir de qualquer local pressionando NVDA+3.
Falar Teclas de Comando

Uma caixa de verificação que quando marcada informa que o NVDA anunciará todas as teclas que não são caracteres pressionadas no teclado. Isto inclui combinações de teclas tais como o control mais outra letra. Pode também configurar esta opção a partir de qualquer local pressionando NVDA+4.

6.2.1.6. Opções do Mouse O diálogo das Opções do Mouse encontra-se no menu Preferências, mais propriamente no item "Opções do Mouse..." e pode ser acessado adicionalmente pressionando NVDA+Control+M. Esta configuração contém as seguintes opções:
Informar Alterações No Mouse

Uma caixa de verificação que quando selecionada informa que o NVDA anunciará a qualquer momento alterações na forma do ponteiro do Mouse . O ponteiro do Mouse altera a sua forma no Windows para fornecer determinada informação tal como quando algo é editável, ou quando alguma coisa está sendo carregada , etc.

Anunciar Texto sob o Mouse

Uma caixa de verificação que quando marcada significa que o NVDA anunciará o texto atual sobre o ponteiro do Mouse , conforme se move pela tela. Isto permite procurar coisas na tela, através do movimento físico do Mouse , antes que precise tentar encontrá-las através da navegação de objetos.

Anunciar Tipo De Controle Quando O mouse entra nele Se esta caixa de verificação estiver ativada, o NVDA anunciará o tipo do objeto quando o Mouse se mover sobre ele.
Tocar sons de coordenadas quando o Mouse é movido Ao marcar esta caixa de verificação faz com que o NVDA reproduza beeps quando o Mouse se mover. Desta forma, o Usuário poderá saber onde é que o Mouse se encontra levando em conta as dimensões da tela.
Brilho controla o volume das coordenadas sonoras Se a caixa de verificação 'tocar as coordenadas de áudio quando o Mouse se mover' estiver marcada, então ao seleccionar esta caixa de verificação significa que o volume dos beeps das coordenadas de áudio é controlado através do brilho da tela que está sobre o Mouse . Isto poderá causar alguns problemas de rendimento no Windows Vista, desta forma,por padrão está desmarcada .
6.2.1.7. Apresentação de Objeto

Encontra-se no menu Preferências no item "Apresentação de Objeto..." ou pressionando NVDA+Control+O, esta caixa de diálogo contém as seguintes opções:

anunciar dicas de Ferramentas

Uma caixa de verificação que quando marcada informa ao NVDA para anunciar sugestões conforme apareçam. Muitas janelas e controles mostram uma pequena mensagem (ou sugestão) quando move o ponteiro do Mouse sobre elas, ou algumas vezes quando move o foco por elas.

Informar Balões de Ajuda

Esta caixa de verificação quando marcada informa ao NVDA para anunciar balões de ajuda conforme apareçam. Os Balões de Ajuda são como pequenas sugestões, mas por norma são maiores em tamanho e estão associados com os eventos do sistema tal como o cabo de rede ser desligado, ou talvez para alertá-lo sobre problemas de segurança do Windows.

Anunciar Teclas de atalho dos Objetos

Quando esta caixa de verificação estiver selecionada, o NVDA incluirá a tecla de atalho que está associada a um determinado objeto ou controle quando o mesmo for anunciado. Por exemplo o menu Arquivo na barra de menus poderá ter a tecla de atalho Alt+F.

Anunciar Descrições de Objetos

Desmarque esta caixa de verificação se não necessita ouvir a descrição anunciada juntamente com os objetos.

Dizer Estado do Objeto primeiro

Quando esta caixa de verificação está marcada, o NVDA anunciará o estado de um objeto antes de qualquer outra informação. Por exemplo, um botão com o nome "reproduzir" e estado "pressionado" será anunciado como "pressionado reproduzir botão" em vez de "reproduzir botão pressionado". Isto é algumas vezes útil se andar com o Tab por objetos do mesmo tipo e estiver realmente só interessado em ver se estão "pressionados", ou "marcados" etc.

Dar Beeps em Atualizações de Barras de Progresso

Uma barra de progresso é um controle que se parece um pouco com uma régua. Conforme uma tarefa está a ser lentamente completada, a barra vai aumentando. Também mostra um valor de percentagem numérico informando-o quando vai avançando. As barras de progresso são apresentadas para coisas como carregamento de páginas da Web, descarregar o seu e-mail, ou processar um arquivo de som etc. Se esta caixa de verificação estiver marcada, o NVDA utilizará beeps para o informar sobre o avanço da barra de progresso. Isto é algumas vezes mais agradável do que ouvir muitos números.

6.2.1.8. Definições do Buffer Virtual

O diálogo das definições do Buffer Virtual pode ser encontrado no menu Preferências, no item "exibidores virtuais...". Alternativamente, pode ser acessado pressionando NVDA+Control+B. Observe que muitas destas definições neste diálogo poderão afetar somente os Buffers Virtuais antigos (usados para o Internet Explorer e Adobe Reader). Em versões futuras do NVDA este problema será resolvido. O diálogo contém as seguintes opções:

Número máximo de Caracteres Por Linha

Este campo define o comprimento máximo de uma linha do Buffer Virtual (em caracteres). Não garante que todas as linhas acabarão tendo esta largura ou inferior, mas a maioria deveria ser assim.

Número Máximo de Linhas Por Página

Pois bem, os Buffers Virtuais não têm realmente páginas. Este campo define a quantidade de linhas ao mover-se quando pressiona page up ou page down no Buffer Virtual.

Usar apresentação da tela quando suportado

Esta caixa de verificação quando selecionada informa ao NVDA para anunciar o texto e informação sobre apresentação no Buffer Virtual para um controle, conforme obtenha o foco, em vez de questionar simplesmente ao Windows pela informação do controle. Por exemplo, ao andar com o Tab para um link denominado "gatos" dirá "link gatos" em vez de "gatos link" porque nos Buffers Virtuais quando navegar, o tipo é sempre anunciado antes do texto. Poderá ser conveniente que em certos casos não tenha esta opção disponível já que isto significa que o controle será anunciado em uma ordem um pouco melhor. Contudo, para alguns navegadores da Web, poderá obter um anúncio mais exato quando isto estiver disponível.

Configurar O Anúncio de Campos Tais Como Links E Títulos

As versões anteriores do NVDA permitiam-lhe utilizar as opções do diálogo do Buffer Virtual para configurar se o NVDA deveria anunciar o tipo de campo assim que entrava nele e informaria isso através de conteúdo da Web. São como exemplos os tipos de campos links, títulos, tabelas e listas. Para configurar isto, por favor localize as novas opções no diálogo das definições de formatação de documentos. Atualmente estas opções não afetam somente o Buffer Virtual, mas sim qualquer documento que possa ter este tipo de campos que são anunciados.

Anunciar apresentação do  exibidor virtual quando o foco muda

Esta opção permite ao modo de foco ser chamado quando o cursor é modificado. Por exemplo, quando numa página Web, se pressionar Tab e encontrar um formulário, caso esta opção esteja marcada, o modo de foco será chamado automaticamente.

Modo de Foco Automático quando o foco muda

Esta opção quando está marcada permite que o NVDA chame e deixe o modo de foco quando andar com as setas direcionais. Por exemplo, se for pressionando Seta Abaixo numa página Web e encontrar uma área de edição, este leitor de tela ativará automaticamente o modo de foco. Se pressionar as setas para sair da área de edição, o NVDA voltará a chamar o modo de navegação.

6.2.1.9.  Formatação de Documentos

Esta caixa de diálogo está no menu Preferências, no item "Formatação de Documentos..." e pode ser Acessada pressionando NVDA+Control+D. Todas as caixas de verificação neste diálogo permitem-lhe configurar qual é o tipo de formatação que pretende ouvir automaticamente quando se mover com o cursor pelos documentos no WordPad ou no Microsoft Word. Por exemplo, se selecionar a caixa de verificação para informar o nome do tipo de letra de cada vez que anda pelo texto com tipos de letra diferentes, o nome do tipo de letra será anunciado. Pode configurar o nome do tipo de letra, tamanho do tipo de letra, atributos do tipo de letra, alinhamento de texto, estilo de texto, tabelas, números de página, números de linha, erros de ortografia, links, títulos, listas e blocos de citação.

6.2.2. Salvar E Restaurar A Configuração

O NVDA Salva as suas configurações em um  arquivo denominado nvda.ini na sua própria pasta. Provavelmente, nunca terá que tocar nele mas para os usuários avançados, este arquivo permite-lhes alterar algumas opções extras que não se encontram nas caixas de diálogo.

O NVDA não Salva automaticamente as suas configurações. Desta forma, se sair do NVDA e reiniciá-lo, estas voltarão ao ponto anterior com os valores originais. Observe que pode alterar esta configuração predefinida através do item "Opções gerais..." que se encontra no menu Preferências. Para guardar as configurações, ou escolhe a qualquer momento o item opções no menu do NVDA ou pressione NVDA+Control+c a partir de qualquer lugar.

Se o NVDA tem um arquivo em um sistema só de leitura, como um CD, não será possível Salvar as suas configurações e o leitor de tela informará.

Se nunca se engana com as suas configuraçõese necessita voltar a trás nas configurações Salvas , pode sempre escolher o item "Restaurar para as Opções Salvas” no menu do NVDA, ou pode pressionar NVDA+Control+r a partir de qualquer local.

7. Navegar Com O NVDA

7.1. Navegando Com O Foco

A forma mais comum de navegar pelo Windows com o NVDA é simplesmente mover-se com os comandos de teclado normais, tais como tab e Shift tab para se mover para a frente e para trás entre controles, pressionar alt para acessar a barra de menus e utilizar as setas para navegar pelos menus e usar alt-tab para se mover entre aplicações em execução. Ao fazer isto, o NVDA anunciará o que está sobre o foco. Quando o NVDA anuncia um objeto (ex.: quando recebe o foco), dirá normalmente o nome do objeto, tipo, estado, valor, descrição, atalho do teclado e informação da posição. Isto é bastante parecido em outros leitores de tela, o que não impede que os tipos de objetos possam ser um pouco diferentes.

Existem alguns comandos de teclado úteis quando se move com o foco:

·         NVDA+Tab – Informa onde está o foco atual;

·         NVDA+B – Anuncia toda a janela em primeiro plano (útil para ler uma caixa de diálogo);

·         NVDA+T – Fala o título da janela atual em primeiro plano.

7.2. Navegar Com O Cursor do Sistema

Quando o foco está sobre um objeto que tem um cursor de edição (cursor do sistema), pode andar com as setas e editar normalmente como pretender. O NVDA anunciará como se mover por Caracter, palavra, linha e também informará a seleção ou não seleção de texto. Quando se mover com o cursor do sistema, a posição do cursor de revisão também será atualizada para se juntar ao mesmo. Pode também ler todo o texto (leitura contínua) com NVDA+Seta Abaixo. Note que isto é diferente do NumpadMais o qual move somente o cursor de revisão, enquanto o primeiro move o cursor do sistema conforme se vai movimentando.

7.3. Navegação De Objetos

Se pretende ter uma visão da aplicação atual ou sistema Operacional , contudo sem mover o foco, pode utilizar a navegação de objetos do NVDA. Isto permite-lhe mover-se de um objeto para outro numa árvore estruturada.

A árvore estruturada dos objetos inicia-se na janela da área de trabalho, ou seja, na raiz da mesma. As primeiras ramificações da árvore são as aplicações atualmente em execução, e como é óbvio, estas aplicações têm as suas próprias ramificações (objetos), que representam diversas partes da aplicação.

A estrutura em árvore pode ser muito grande, mas permite-lhe navegar numa ordem lógica, de forma a conseguir encontrar a informação que necessita. Esta estrutura não tem nada a ver com a forma em que os objetos estão dispostos na tela, mas sim como estão relacionados logicamente. Isto poderá demorar algum tempo até que comece a utilizar, mas uma vez que compreenda isto, deverá conseguir mover-se com muita facilidade. Atualmente o NVDA não pode navegar pelos objetos através do layout baseado na tela porque não tem nenhum driver de intercepção de vídeo ou display hooks. O NVDA não escolhe interagir com a tela porque deve permitir acima de tudo mais clareza e mais percepção lógica de uma aplicação e do sistema operativo, antes de tentar captar a disposição do layout da tela. Nem todas as aplicações têm uma estrutura em árvore perfeita, mas isto depende do programador dessa aplicação em particular. A maioria está correta e certamente, grande parte das caixas de diálogo e aplicações muito estruturadas parecem estar bem.

Para navegar através de objetos utilize as teclas seguintes:

·         NVDA+Numpad5: Anuncia o objeto actual;

·         NVDA+Numpad8: Move para o objeto ascendente (Movendo na direção da raiz da árvore);

·         NVDA+Numpad4: Move para o objeto anterior (o objecto imediatamente anterior a este no mesmo nível);

·         NVDA+Numpad6: Move para o objeto seguinte (O objeto imediatamente seguinte a este no mesmo nível);

·         NVDA+Numpad2: Move para o primeiro objeto descendente (O primeiro ramo que sai deste objecto);

·         NVDA+Shift+Numpad4: Move para o objeto anterior no fluxo (Atravessa automaticamente fronteiras de ascendentes e descendentes até encontrar o anterior);

·         NVDA+Shift+Numpad6: Move para o próximo objeto no fluxo (Atravessa automaticamente fronteiras de ascendentes e descendentes até encontrar o próximo);

·         NVDA+NumpadMenos: Move para o foco;

·         NVDA+Numpad Enter: Ativar o objeto atual (ex. pressionar Enter / clique / duplo clique);

·         NVDA+NumpadBarra: Move o Mouse para o objeto atual;

·         NVDA+NumpadAsterisco: Move para o Mouse ;

·         NVDA+Shift+Numpad5: anunciar as dimensões atuais dos objetos referentes a tela;

·         NVDA+NumpadMais: Leitura contínua de todos os objetos – Começa a ler do objeto atual, prosseguindo pelos próximos no fluxo.

Nota: As teclas do teclado numérico requerem que o NumLock esteja desativado para que possam trabalhar adequadamente. Quando se mover com o foco, o objeto de navegação atual altera-se juntamente com ele. Contudo, se pretende bloquear a navegação de objeto para um objeto específico, para que este não se altere com o foco, pode desativar o foco a Mover o Objeto de navegação com NVDA+7.

7.4. Rever Texto No Objeto Atual

O NVDA permite-lhe ler o conteúdo da navegação de objetos atual, por caracteres, palavras ou linhas. Isto é muito útil na consola do Dos do Windows e noutros locais onde há um cursor do sistema limitado ou inexistente.

·         Shift+Numpad7: Primeira linha;

·         Numpad7: Linha anterior;

·         Numpad8: Linha atual;

·         Numpad9: Linha seguinte;

·         Shift+Numpad9: Última linha;

·         Numpad4: Palavra anterior;

·         Numpad5: Palavra atual;

·         Numpad6: Palavra seguinte;

·         Shift+Numpad1: Início da linha;

·         Numpad1: Caracter anterior;

·         Numpad2: Caracter actual;

·         Numpad3: Caracter seguinte;

·         Shift+Numpad3: Fim da linha;

·         NumpadMais: Leitura contínua de todo o texto;

Nota: As teclas do teclado numérico requerem que o NumLock esteja desativado para que possam funcionar adequadamente.

Quando um objeto tem um cursor de edição (cursor do sistema) e move-se, o cursor de revisão é também atualizado para essa posição. Se pretende que o cursor de navegação permaneça onde está e que não se mova com o cursor do sistema, pode desativar o "Cursor do sistema Move O Cursor de Navegação" ao pressionar NVDA+6.

É possível copiar partes de um conteúdo que está a rever. Por exemplo, poderá querer copiar um endereço da Internet que alguém lhe enviou num texto num programa de conversação. Para copiar o conteúdo, comece por mover o cursor de navegação até o primeiro caracter do texto que pretende copiar e pressione NVDA+F9. Por fim, mova-se para o final do texto e pressione NVDA+F10. O conteúdo será copiado para a área de transferência, o que lhe possibilita copiar o conteúdo onde desejar . Atente que o programa que está utilizando já lhe possibilita mover-se ao longo do texto utilizando o cursor do sistema e copiar através do comando normal Control+C, o que lhe trará melhores resultados.

7.5. navegando Com O Mouse Quando mover o mouse , o NVDA informará por predefinição o texto que se encontra diretamente sobre o ponteiro deste periférico, assim que este se mova pelo texto. Quando suportado, este leitor de tela lerá o conteúdo de parágrafos de texto, contudo em alguns controles só poderá ler por linha.

O NVDA também pode ser configurado para anunciar o tipo de controle ou o objeto onde está atualmente o Mouse conforme se movimente (ex. lista, botão etc.), o que é muito útil para usuários totalmente cegos quando algumas vezes o texto não for suficiente.

O NVDA fornece um modo para que os Usuários compreendam onde está o mouse no que se refere às dimensões da tela através da reprodução das coordenadas atuais do mouse com beeps audíveis. Quanto mais alto estiver o mouse na tela, mais agudo será o tom dos beeps. Quanto mais à esquerda ou à direita estiver o mouse na tela, mais parecerá que o som está à esquerda ou à direita (assumindo-se que o usuário tem umas colunas stereo).

Estas características adicionais do Mouse não se encontram ativadas por padrão no NVDA, mas se desejar utilizar esses recursos , pode configurá-las a partir do diálogo das opções do mouse, que se encontram no item Preferências no menu do NVDA.

8. Buffers Virtuais

Os documentos complexos só de leitura, tais como páginas Web, são representados no NVDA em um  Buffer Virtual, (exibidor virtual).

Um Buffer Virtual é uma representação plana textual do documento, que permite ao usuário acesso  ao texto e aos   campos existentes no documento.  além de conseguir mover-se, também pode pressionar Enter ou espaço em links e botões para os ativar, pressionar Enter ou espaço em áreas de edição para editar o conteúdo respectivo, carregar no Enter ou no espaço em caixas de verificação ou em botões de opção para os selecionar e pressione Enter ou espaço em caixas de combinação para modificar imediatamente a sua seleção.

Quando está num Buffer Virtual, existem dois modos de interação. O modo de navegação é onde se pode mover pelo texto utilizando um cursor virtual. O modo de foco é onde pode interagir diretamente com um controle (área de edição, caixa de combinação, botões de opção) utilizando adequadamente as teclas que usaria normalmente. Quando estiver no modo de navegação, o NVDA ativará automaticamente o modo de foco se andar com o tab ou clicar num controle em particular que solicite isso. Também pode evocar manualmente o modo de foco com NVDA+Espaço. Se o NVDA alternou para o modo de foco automaticamente por causa de algum Tab ou clique, poderá voltar ao ponto anterior para o modo de navegação uma vez que pressionou tab ou clique algumas vezes em algo que não precisava disto. Contudo, se evocou o modo de foco com NVDA+Espaço, ele permanecerá assim até que volte manualmente ao modo de navegação. Quando estiver no modo de foco, pode voltar para o modo de navegação a qualquer momento pressionando Escape ou NVDA+Espaço. O diálogo das definições do Buffer Virtual no menu Preferências do menu do NVDA permite-lhe desactivar automaticamente o alternar entre os modos de navegação e foco.

Atualmente o NVDA utiliza os Buffers Virtuais para ler documentos no Mozilla Firefox, Microsoft Internet Explorer e Adobe Reader.

Note que os Buffers Virtuais usados para o Internet Explorer e Adobe Reader são muito básicos e requerem muito mais tempo para carregar. Estes Buffers Virtuais antigos nem sequer tratam muito bem conteúdo dinâmico, nem fornecem suporte à Lista de Links ou a algumas teclas de navegação rápida do Buffer Virtual. Para uma melhor performance e facilidade de uso, teste o Mozilla Firefox para navegar na Web.

8.1. Navegação Por Caracter

Para uma navegação rápida, o NVDA também fornece teclas de caracter simples para saltar para determinados campos no Buffer Virtual.

Somente a letra salta para o próximo campo, acompanhada com a tecla Shift salta para o campo anterior.

Teclas suportadas pelos Buffers Virtuais antigos e novos:

·         H: Título;

·         L: Lista;

·         I: Item de lista;

·         T: Tabela;

·         K: Link;

·         P: Parágrafo;

·         F: Campo de formulário.

Teclas não suportadas pelo Buffer Virtual antigo:

·         U: Link não visitado;

·         V: Link visitado;

·         E: Área de edição;

·         B: Botão;

·         X: Caixa de verificação;

·         C: Caixa de combinação;

·         R: Botão de opção;

·         Q: Bloco de citação;

·         S: Separador;

·         M: Frame;

·         G: Gráfico;

·         1 até 6: Títulos de nível 1 até 6 respectivamente.

8.2. Formulários E OS Modos de Foco e Navegação

Algumas vezes precisará interagir com campos de formulário alternando entre o modo de foco e o de navegação,  com as teclas do Buffer Virtual. O modo de foco foi criado para este propósito e pode ser ativado e desativado com NVDA+espaço.

8.3. A Lista de Links

Para Acessar à lista de links para o Buffer Virtual atual, pressione NVDA+f7. A partir do diálogo que será exibido, , pode selecionar o link no qual está interessado e então, ou move-se para ele, ou ativa-o utilizando os botões fornecidos.
